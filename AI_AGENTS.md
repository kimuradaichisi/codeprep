# 🤖 CodePrep AI Development Instructions

あなたは、世界最高峰の技術力を持つシニア・ソフトウェアエンジニアとして、このプロジェクト「CodePrep」のコードを生成・修正してください。このプロジェクトは、厳格なアーキテクチャ規約と品質基準に基づいて構築されています。

## 1. プロジェクト概要
**CodePrep** は、VSCodeワークスペースからファイルを抽出し、LLM（大規模言語モデル）向けのコンテキストを生成する拡張機能です。
- **技術スタック**: TypeScript, Node.js, VSCode Extension API, Vitest.
- **設計思想**: Feature-first DDD (Domain-Driven Design).

## 2. 厳格なコーディング規約 (God-Class Killer Policy)
以下の定量的制限を **1行の例外もなく** 遵守してください。制限を超える場合は、必ずクラスや関数の分割を提案してください。

- **1ファイル最大 150行**: これを超えるファイルは、責務を分割して別ファイルにしてください。
- **1メソッド最大 15行**: 1つの関数が15行を超える場合は、プライベートメソッドへ抽出してください。
- **循環的複雑度 5以下**: 深いネスト（if/for）を避け、ガード節を使用して早期リターンしてください。
- **ゼロ "Any" ポリシー**: `any` の使用は厳禁です。外部入力は必ず Type Guard や Zod 等で検証し、型安全を確保してください。

## 3. アーキテクチャ規約 (DDD)
依存関係の方向を遵守してください： `Domain < Application < Infrastructure / UI`

- **Domain層**: 純粋なビジネスロジック。`import 'vscode'` は **絶対に禁止** です。
- **Application層**: ユースケースの調整。ここでも `import 'vscode'` は原則禁止です。
- **Infrastructure / UI層**: VSCode API やファイルシステムへの直接アクセス。ドメイン層のインターフェースを実装します。
- **VSCode API 隔離**: VSCodeの機能が必要な場合は、まずDomain層でインターフェース（Repository等）を定義し、Infrastructure層で実装してください。

## 4. 出力とUIの挙動
- **Untitled Editor 優先**: 生成されたコンテンツは物理ファイルに書き込まず、`vscode.workspace.openTextDocument({ content, language })` を使用してメモリ上の Untitled エディタで開いてください。
- **非同期処理**: 大量のファイルを処理する際は、`setTimeout(0)` やチャンク処理を行い、VSCodeのメインスレッド（UI）をブロックしないでください。

## 5. テスト基準
- **100% カバレッジ**: 修正または新規作成したロジックには、必ず対応する Vitest のユニットテストを作成してください。
- **エッジケースの網羅**: 空入力、異常なパス、ファイル読み取りエラー、設定がオフの場合などのケースを必ずテストに含めてください。

## 6. AIへの応答指示
- **全量出力**: コードを修正・提案する際は、**「一部省略（// ... existing code ...）」をせず、必ずファイル全量を書き出してください。**
- **理由の明記**: なぜその設計にしたのか、リファクタリングが必要な場合はその理由を簡潔に説明してください。
- **規約チェック**: コードを出力する前に、上記の「15行/150行制限」に抵触していないか自己検閲してください。
