# Implementation Audit Report: GitUtils & SelectionService

## 1. 監査結果サマリー
現状の実装には、エッジケースにおける動作不良と、外部環境の変化に対する脆弱性が含まれている。特にファイルパスのパース処理は「壊れやすい（Brittle）」状態である。

## 2. 特定された脆弱性・バグ
### 2.1. GitUtils: パスパースの不具合
- **問題**: `line.trim().split(' ')` を使用してパスを取得している。
- **再現条件**: ファイル名にスペースが含まれる場合（例: `My Document.txt`）。
- **結果**: `split` によりパスが分割され、末尾の断片のみが取得されるため、正しいパスを選択できない。
- **修正案**: 固定幅（最初の3文字以降）を取得するか、正規表現による正確なパースが必要。

### 2.2. SelectionService: プリセット名のバリデーション欠如
- **問題**: `savePreset(name: string)` において、`name` の妥当性チェックがない。
- **リスク**: 空文字、OSのファイルシステムで禁止されている文字、あるいは非常に長い文字列が渡された場合、`vscode.Memento` のストレージ制限やUI表示の崩壊を招く。

## 3. テスト戦略
### 3.1. ユニットテスト導入
- **フレームワーク**: `vitest` (TypeScriptとの親和性と高速な実行速度のため)
- **対象**:
    - `GitUtils`: モック化した `child_process` 出力に対するパース精度の検証
    - `SelectionService`: `vscode.Memento` のモックを使用した永続化ロジックの検証

### 3.2. 境界条件テストケース
- [ ] Git未インストール環境での `GitUtils.getModifiedFiles` の挙動
- [ ] スペース、日本語、特殊記号を含むファイルパスのパース
- [ ] 大規模な差分（1000ファイル以上）発生時のパフォーマンス
- [ ] 同名のプリセット保存による上書き確認
