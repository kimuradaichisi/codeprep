# E2E テストシナリオ: UI可視性と初期化の検証

## 1. 目的
「サイドバーからメニューが消えた」という事象の再発を防止するため、拡張機能の起動直後および状態変化時の UI 状態を自動検証する。

## 2. テストケース

### 2.1. ワークスペース未オープン時の初期化 (Critical)
- **前提**: VSCode でフォルダが開かれていない。
- **手順**: 拡張機能をアクティベートする。
- **期待結果**:
  - `codeprep.fileTree` ビューが登録されていること。
  - `codeprep.selectionEmpty` コンテキストキーが `true` であること。
  - Welcome View の内容（[すべて選択] 等）が理論上アクセス可能であること。

### 2.2. ボタンのカスタマイズ反映
- **前提**: 設定 `codeprep.visibleButtons` に特定のコマンドのみを含める。
- **手順**: 設定を更新する。
- **期待結果**:
  - 設定に含まれるボタンのコンテキストキー（例: `codeprep.showSelectAll`）が `true` になること。
  - 設定に含まれないボタンのコンテキストキーが `false` になること。

### 2.3. フォルダ追加時の動的更新
- **前提**: 空の状態で起動。
- **手順**: フォルダをワークスペースに追加する。
- **期待結果**:
  - `FileTreeProvider` のルートが更新されること。
  - ファイルツリーの描画が試行されること。
