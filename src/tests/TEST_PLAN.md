# 🧪 CodePrep テスト計画書

## 1. 目的
本ドキュメントは、VSCode拡張機能「CodePrep」の品質を保証するためのテスト戦略と計画を定義する。
ユニットテストによるロジック検証に加え、実際のVSCode環境での挙動を保証するE2Eテスト（End-to-End）を実施する。

## 2. テスト範囲
### 2.1 ユニットテスト (Unit Tests)
- **対象**: `src/services`, `src/engines`, `src/utils` 配下の純粋なロジック。
- **目的**: 外部依存（VSCode API）をモックし、計算ロジックや状態遷移の正当性を検証する。
- **ツール**: Vitest

### 2.2 E2Eテスト (Integration/E2E Tests)
- **対象**: `extension.ts`, `providers`, `contributes` (コマンド、メニュー)。
- **目的**: 実際のVSCodeインスタンスを起動し、コマンドの登録、TreeViewの表示、クリップボード操作が正常に行われるか検証する。
- **ツール**: @vscode/test-electron, Mocha

## 3. テストシナリオ
### 3.1 正常系シナリオ
- **TS-001: 拡張機能の有効化**
    - 期待結果: 拡張機能が正常にロードされ、`codeprep.fileTree` ビューが表示されること。
- **TS-002: ファイル選択とプロンプト生成**
    - 手順: ファイルをTreeViewで選択し、`codeprep.generate` コマンドを実行。
    - 期待結果: 生成されたプロンプトがクリップボードにコピーされ、通知が表示されること。
- **TS-003: エンジン切り替え**
    - 手順: 設定で `useNativeEngine` を切り替え。
    - 期待結果: 選択したエンジンが正しく使用されること。

### 3.2 異常系・境界系シナリオ
- **TS-101: ファイル未選択での生成**
    - 期待結果: 警告メッセージが表示され、処理が中断されること。
- **TS-102: 無効なGitリポジトリでの操作**
    - 期待結果: エラーハンドリングが行われ、ユーザーに通知されること。

## 4. 実行環境
- **OS**: Windows / Linux / macOS (CI環境含む)
- **Node.js**: v20以上
- **VSCode**: ^1.84.0

## 5. 合否判定基準
- 全てのユニットテストおよびE2Eテストがパスすること。
- テストカバレッジが主要ロジックにおいて80%以上であること。
